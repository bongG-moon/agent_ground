# 포털 UI 검증 체크리스트

## 정보

- 제목만 읽어도 자산 목적을 알 수 있는가?
- 첫 화면에서 입력과 출력의 수, 핵심 타입, 다음 연결을 알 수 있는가?
- 공용 자산과 특정 Flow 내부 구성 요소가 구분되는가?
- Component 수가 내부 실행 노드 수를 포함하지 않는가?
- 내부 노드의 상세·코드 화면이 소유 Flow로 돌아가는가?
- 상태와 마지막 검증 환경이 실제 manifest와 일치하는가?

## 링크와 코드

- 계약 버튼이 같은 페이지의 존재하는 anchor로 이동하는가?
- 코드 버튼이 raw `.py`나 사용자 절대경로를 열지 않는가?
- 코드의 `<`, `>`, `&`, 따옴표가 HTML로 안전하게 escape되는가?
- 줄 번호, 복사, 가로 스크롤과 keyboard focus가 동작하는가?
- 홈, 이전, 상위 목록 링크가 모두 존재하는가?

## 레이아웃

- 1280px에서 교육 단계 문장이 두세 글자 단위로 잘리지 않는가?
- 768px에서 grid가 자연스럽게 줄고 nav와 버튼이 겹치지 않는가?
- 360px에서 좌우 페이지 overflow가 생기지 않는가?
- 넓은 표가 wrapper 안에서만 스크롤되는가?
- 긴 code, JSON, URL이 카드 폭을 강제로 늘리지 않는가?
- sticky header가 표 내용과 겹치거나 잘리지 않는가?

## 접근성

- skip link와 visible focus가 있는가?
- 버튼과 링크에 의미 있는 텍스트 또는 aria-label이 있는가?
- 색만으로 상태를 구분하지 않는가?
- 모바일 touch target이 충분한가?
- `prefers-reduced-motion` 사용자가 불필요한 animation을 피할 수 있는가?
