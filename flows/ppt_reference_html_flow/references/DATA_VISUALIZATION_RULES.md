# 데이터 입력과 시각화 선택 규칙

이 문서는 `ppt_reference_html_flow`가 데이터셋을 어떻게 받고, 어떤 조건에서 표·KPI·차트를 선택하는지 정의합니다. 사용자가 지정한 표현 방식이 있으면 우선하되 데이터 구조나 발표 목적과 모순되면 경고하고 안전한 표현으로 바꿉니다.

## 1. 데이터셋 입력 계약

```json
{
  "dataset_id": "monthly_sales",
  "title": "월별 매출",
  "description": "2026년 상반기 채널별 확정 매출",
  "source": "사내 DW SALES_MART",
  "period": "2026-01~2026-06",
  "preferred_visual": "auto",
  "columns": [
    {
      "name": "month",
      "label": "월",
      "semantic_type": "temporal",
      "unit": "",
      "format": "YYYY-MM"
    },
    {
      "name": "sales",
      "label": "매출",
      "semantic_type": "quantitative",
      "unit": "억원",
      "format": ",.1f"
    },
    {
      "name": "channel",
      "label": "채널",
      "semantic_type": "nominal",
      "unit": "",
      "format": ""
    }
  ],
  "rows": [
    {"month": "2026-01", "sales": 121.4, "channel": "온라인"}
  ]
}
```

### 의미 타입

| 타입 | 의미 | 예시 |
| --- | --- | --- |
| `quantitative` | 계산 가능한 수치 | 매출, 수량, 오류율 |
| `temporal` | 날짜·시간 순서 | 일자, 월, 분기 |
| `ordinal` | 순서가 있는 범주 | 등급, 우선순위 |
| `nominal` | 순서가 없는 범주 | 부서, 채널, 제품 |
| `text` | 자유 서술 | 의견, 원인, 조치 |
| `identifier` | 집계하면 안 되는 식별자 | 주문번호, 문서 ID |

숫자로 보이는 코드와 ID는 `quantitative`로 지정하지 않습니다. 날짜 문자열은 `temporal`과 실제 형식을 함께 제공합니다.

## 2. 사용자 선택값

`preferred_visual`은 다음 값을 허용합니다.

- `auto`
- `kpi`
- `table`
- `bar`
- `line`
- `stacked_bar`
- `scatter`
- `histogram`

사용자 선택을 변경하는 경우 `warnings`에 원래 값, 변경값과 이유를 남깁니다.

0.1.0 Renderer는 `kpi`, `table`, `bar`, `line`, `scatter`를 실제 시각 요소로 만듭니다. `stacked_bar`와 `histogram` 입력은 계약에는 허용하지만, 임의 집계·구간 계산을 하지 않고 원본 표로 안전하게 변경하며 경고를 남깁니다.

## 3. 표현 선택 의사결정표

| 사용자 목적 | 데이터 조건 | 기본 표현 | 보완 조건 |
| --- | --- | --- | --- |
| 핵심 결과를 빠르게 전달 | 의미 있는 지표 1~4개 | KPI 카드 | 비교 기준과 단위를 함께 표시 |
| 항목별 정확한 값 확인 | 열이 여러 개이거나 행 단위 조회 | 표 | 중요 열만 우선 표시, 원본 다운로드 경로 고려 |
| 범주별 크기 또는 순위 비교 | 범주 2~12개 + 수치 1개 | 막대그래프 | 긴 범주명은 가로 막대, 값 기준 정렬 |
| 시간 추세 확인 | 시간 필드 + 수치 | 선그래프 | 시간순 정렬, 누락 기간 표시 |
| 구성비 변화 비교 | 범주 + 그룹 + 수치 | 100% 누적 막대 | 그룹 수 제한, 전체 합계도 별도 제공 |
| 두 수치의 관계 확인 | 수치 필드 2개 | 산점도 | 점 수, 이상치, 상관관계 과해석 주의 |
| 수치 분포 확인 | 수치 1개, 충분한 행 수 | 히스토그램 | 구간 기준과 누락값 수 표시 |
| 단계와 책임 설명 | 순서가 있는 업무 단계 | 프로세스/타임라인 | 실제 데이터 차트로 위장하지 않음 |
| 한 문장으로 충분한 결과 | 데이터가 매우 적음 | 수치 강조 또는 설명 | 불필요한 차트를 만들지 않음 |

## 4. 표 선택 규칙

표를 우선하는 경우:

- 사용자가 개별 값, 코드, 날짜를 정확히 읽어야 함
- 수치와 설명 열이 함께 있어 차트로 의미가 손실됨
- 비교 대상이 적지만 지표가 여러 개임
- 원본 근거를 발표 자료 안에서 확인해야 함

표 제한:

- 한 슬라이드 기본 표시 행은 12행 이하
- 열은 핵심 6개 이하를 권장
- 초과 데이터는 다음 슬라이드로 분할하거나 요약표와 원본 링크로 나눔
- 숫자는 오른쪽 정렬, 동일 단위와 소수 자릿수 적용
- 헤더는 짧고 명확한 사용자용 label 사용
- 행·열 잘림이 예상되면 Quality Gate에서 경고 또는 차단

## 5. KPI 규칙

- 한 슬라이드에 1~4개만 사용합니다.
- 값, 단위, 기준 기간과 비교 대상을 함께 표시합니다.
- 증감률은 기준값과 현재값이 모두 있을 때만 계산합니다.
- 0으로 나누는 증감률은 생성하지 않고 `undefined_baseline` 경고를 남깁니다.
- 좋은 변화와 나쁜 변화를 색상만으로 구분하지 않고 화살표와 설명을 함께 사용합니다.

## 6. 막대그래프 규칙

- 범주 비교와 순위에 사용합니다.
- 정량 축은 특별한 이유가 없으면 0부터 시작합니다.
- 범주명 길이가 길거나 범주 수가 많으면 가로 막대를 선택합니다.
- 범주 수는 기본 12개 이하로 제한합니다.
- 자동 선택에서는 범주 12개 이하일 때 막대그래프를 권장합니다. 더 많은 범주는 임의의 상위 N·`기타` 집계를 만들지 않고 표 또는 분할 입력을 권장합니다.
- 그룹 막대는 시리즈 4개 이하를 권장합니다.
- 값 레이블과 단위를 제공하고 장식용 3D 효과를 사용하지 않습니다.

## 7. 선그래프 규칙

- 시간 또는 실제 순서가 있는 축에 사용합니다.
- 0.1.0은 원본 행 순서를 보존하므로 입력 dataset을 시간 오름차순으로 정렬해 넣습니다.
- 시리즈는 기본 4개 이하를 권장합니다.
- 색상 외에 점 표식, 선 형태, 직접 레이블을 사용합니다.
- 누락 기간을 0으로 임의 치환하지 않습니다.
- 값이 없는 구간은 끊긴 선 또는 명시적 누락으로 표현합니다.
- 급격한 변화가 있으면 원본 값과 기간을 검증한 뒤 설명합니다.

## 8. 누적 막대 규칙

> 0.1.0 실행 제한: 누적 계산을 임의 수행하지 않으며 `stacked_bar`는 원본 표로 변경하고 경고합니다. 아래 규칙은 후속 Renderer 확장 시 적용할 기준입니다.

- 전체 대비 구성비나 그룹별 구성 변화를 비교할 때 사용합니다.
- 비율 비교가 목적이면 100% 누적을 사용합니다.
- 절대 규모가 중요하면 일반 누적 막대와 합계 레이블을 사용합니다.
- 구성 항목이 너무 많으면 작은 항목을 `기타`로 묶습니다.
- 음수와 양수가 섞인 값은 별도 설명 없이 누적하지 않습니다.

## 9. 산점도 규칙

- x와 y 모두 quantitative일 때 사용합니다.
- 식별자, 시간 순서 또는 범주 코드를 숫자 축에 넣지 않습니다.
- 점이 겹치면 투명도 또는 크기를 조절하고 전체 행 수를 표시합니다.
- 상관관계를 인과관계로 표현하지 않습니다.
- 추세선은 충분한 데이터와 계산 근거가 있을 때만 추가합니다.
- 이상치 기준을 사용자가 주었거나 계산 규칙이 명확할 때만 이상치로 표시합니다.

## 10. 히스토그램 규칙

> 0.1.0 실행 제한: bin 경계와 집계 근거가 없는 히스토그램을 만들지 않으며 원본 표로 변경하고 경고합니다. 아래 규칙은 후속 Renderer 확장 시 적용할 기준입니다.

- 연속형 수치의 분포를 설명할 때 사용합니다.
- 행이 매우 적으면 점 그래프나 표를 검토합니다.
- 구간 수와 구간 폭을 `calculation_note`에 기록합니다.
- 누락값, 최솟값, 최댓값과 행 수를 함께 제공합니다.
- 막대 사이 간격은 범주 막대보다 작게 표시합니다.

## 11. 파이·도넛 차트 정책

기본 허용 목록에는 포함하지 않습니다. 구성비는 100% 누적 막대가 항목 간 비교와 접근성 설명에 더 적합하기 때문입니다. 사용자가 도넛 차트를 명시적으로 요청하면 다음 조건을 모두 만족할 때만 확장 후보로 검토합니다.

- 전체 합이 명확함
- 항목이 5개 이하임
- 음수가 없음
- 각 항목의 비율과 레이블을 직접 표시함
- 동일 내용을 표 또는 텍스트로도 제공함

## 12. 계산과 집계 규칙

- 0.1.0은 Flow 안에서 `sum`, `avg`, 비율, 순위, binning을 새로 계산하지 않습니다. 차트에 필요한 확정 계산값을 dataset 행으로 먼저 제공합니다.
- 후속 버전에서 계산을 추가할 때는 모든 계산을 `data_views[].aggregation` 또는 `calculation_note`로 추적하고 `sum`, `avg`, `min`, `max`, `count`, `none`만 기본 허용합니다.
- 평균을 도입할 때는 단순 평균인지 가중 평균인지 구분합니다.
- 퍼센트 단위를 다시 100배 하지 않도록 원본 단위를 확인합니다.
- 서로 다른 단위의 값을 하나의 축에 섞지 않습니다.
- 반올림은 표시 단계에서만 수행하고 원본 계산값을 변경하지 않습니다.
- 날짜와 금액 형식은 데이터셋의 `format`을 우선합니다.

## 13. 데이터 품질 경고

다음 조건은 계획 생성 전에 확인합니다.

- 필수 필드 누락
- 숫자 또는 날짜 파싱 실패
- 동일 ID 중복
- 단위 또는 출처 누락
- 빈 데이터셋
- 행 수가 지나치게 많아 모델 입력 제한 초과
- 혼합 단위
- 분모가 0인 비율
- 요청 기간 밖의 데이터

경고가 있어도 결과를 만들 수 있는 경우 `warning`, 사실 왜곡 위험이 있으면 `error`로 처리합니다.

## 14. 접근성 규칙

- 차트의 핵심 결론을 한 문장으로 제공합니다.
- 필요한 경우 원본 값을 표 또는 상세 설명으로 제공합니다.
- 시리즈는 색상만으로 구분하지 않습니다.
- 일반 텍스트는 최소 4.5:1, 의미 있는 그래픽은 최소 3:1 대비를 목표로 합니다.
- SVG 차트에는 전체 설명과 가능한 경우 mark description을 제공합니다.
- Canvas 차트를 사용할 경우 `role=img`, `aria-label`과 fallback 내용을 제공합니다.

## 15. `auto` 선택 예시

```text
month(temporal) + sales(quantitative)
-> 시간 변화가 목적
-> line

department(nominal) + incidents(quantitative)
-> 부서별 크기와 순위가 목적
-> horizontal bar

order_id(identifier) + status(nominal) + amount(quantitative)
-> 개별 건 확인이 목적
-> table

lead_time(quantitative) 500행
-> 분포와 이상 구간 확인이 목적
-> 0.1.0은 table, 후속 binning 지원 시 histogram
```
