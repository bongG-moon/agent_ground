# MSG 업로드 가이드

## 권장 순서

1. 분석 대상 메일을 Outlook `.msg` 형식으로 저장합니다.
2. 사내 보안 정책상 업로드 가능한 메일인지 확인합니다.
3. `01 MSG 본문·첨부파일 분해`의 Outlook MSG 파일 입력에 하나 이상의 `.msg`를 올립니다.
4. 사내 DRM 구현과 두 Language Model 설정이 완료됐는지 확인합니다.
5. Chat Input에 원하는 정리 기준을 입력합니다.

## 예시 요청

```text
업로드한 MSG별로 본문과 첨부파일을 검토해 다음 주 회의 전에 해야 할 일을 정리해줘.
담당자와 기한은 원문에 명시된 경우에만 쓰고, 메일이나 첨부 간 수치가 다르면 충돌로 표시해줘.
```

## 업로드 묶음 예시

```text
review_request.msg
supplier_followup.msg
meeting_result.msg
```

실제 메일이나 첨부파일을 저장소 샘플에 넣지 마세요. 자료는 실행 시에만 업로드합니다.

## 사전 확인

- `.eml`이 아니라 `.msg`인지 확인합니다.
- MSG 파일 자체가 DRM 보호된 경우 이 Flow에 넣기 전에 승인된 방식으로 해제합니다.
- 인라인 로고까지 필요할 때만 `인라인 이미지 포함`을 켭니다.
- 내장 MSG 첨부는 현재 자동 재귀 분해하지 않습니다.
