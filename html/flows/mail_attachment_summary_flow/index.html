EWS·DRM Outlook 메일 요약 Flow

사내 EWS로 메일을 읽고 일반 문서는 로컬 파싱, 보호 문서는 DRM API, JPG/JPEG는 vLLM Vision 모델로 처리하며 미지원 형식은 건너뛴 뒤 전체 요약을 생성합니다.

Flow JSON 열기사용 가이드 열기
현재 상태는 user_testing입니다.정적 schema와 격리 계약 검증 범위는 manifest에 기록되어 있으며, 실제 사내 endpoint·자격증명·업무 파일을 사용한 확인과 사용자 완료 승인은 별도입니다.

Configuration

첫 실행 전에 입력할 값

  • Configure EWS URL, mailbox, AD account, password, and the internal Nexus settings used by the actual environment.
  • Prefer enabling TLS verification with the internal CA bundle after confirming the current EWS certificate chain.
  • Choose auto, always DRM, or DRM bypass processing mode for attachments.
  • Configure the approved DRM text API URL, Bearer token, and employee number when DRM calls may occur.
  • For an HTTP-only closed test network, explicitly enable insecure HTTP after security approval.
  • Confirm the DRM API text extraction coverage for PDF, Office, HWP, text, CSV, and non-JPEG image attachments.
  • Configure the JPG/JPEG Language Model node with an approved vLLM-compatible multimodal model.
  • Select the same approved internal or local model in both summary Language Model nodes.
